# 【パート1】シンプル版おみくじ

## 完成イメージ

- ボタンを押す → 「結果：？？？」が「結果：（引いた結果）」に変わる
- 結果は **6種類** からランダムで1つ選ばれる

配布画像 `omikuji01.png`（デフォルト画面）と `omikuji02.png`（ボタンを押した後）を見て、動きをイメージしよう。

## 仕様

### ① ウィンドウ

| 項目 | 値 |
|------|-----|
| タイトル | `おみくじアプリ` |
| サイズ | `350x250` |

### ② 画面に置く部品（ウィジェット）

`pack()` を使って、上から順に並べよう。

1. **タイトルのラベル** … 文字「おみくじアプリ」／フォントサイズ 16
2. **説明のラベル** … 文字「ボタンを押して、おみくじを引こう！」／フォントサイズ 12
3. **ボタン** … 文字「おみくじを引く」／フォントサイズ 14
4. **結果のラベル** … 文字「結果：？？？」／フォントサイズ 18

> 💡 ヒント：ラベルもボタンも、作っただけでは画面に出ない。`pack()` で配置するのを忘れずに（lesson10でやったね）。

### ③ ボタンを押したときの動き

ボタンを押すと、おみくじの結果がランダムで選ばれて、結果ラベルに表示される。

このために **関数** を1つ作って、ボタンの `command` に結びつけよう（lesson11でやった `command=関数名` の形）。

関数の中でやること：
1. 6種類の結果からランダムで1つ選ぶ
2. 選んだ結果を、結果ラベルに `config` で表示する（「結果：〇〇」の形）

#### 🆕 今回の新しい道具：`random.choice()`

lesson06のおみくじでは `random.randint()` で数字を出して、`if`で「1なら大吉、2なら…」と振り分けたよね。
今回はもっとラクな書き方を使う。

```python
import random

fortunes = ["大吉", "中吉", "小吉", "吉", "末吉", "凶"]
result = random.choice(fortunes)   # リストの中からランダムで1つ選ぶ
```

`random.choice(リスト)` は、**リストの中から1個だけランダムに取り出す**命令。
`randint`＋長い`if/elif`が、たった1行で書けてしまう。便利！

> 💡 結果ラベルへの表示は `result_label.config(text=f"結果：{result}")` のように、f文字列を使うときれいに書けるよ。

### 〔チェックポイント②〕
書けたら実行！ ボタンを押すたびに結果が変わればシンプル版は完成！🎉

# 【パート2】背景画像つきリッチ版

シンプル版を、背景画像が入ったかっこいいアプリに進化させよう。
**ここが今日の山場！** 新しい道具が3つ出てきます。

## 完成イメージ

- ウィンドウいっぱいに背景画像（神社の絵）が表示される
- その上に、タイトル・説明・ボタン・結果が**好きな位置に**置かれている

## 仕様

### ① ウィンドウ

| 項目 | 値 |
|------|-----|
| タイトル | `おみくじアプリ` |
| サイズ | `640x480` |
| サイズ変更 | できないようにする（`root.resizable(False, False)`） |

### ② 背景画像を表示する

#### 🆕 新しい道具その1：`PhotoImage`（画像の読み込み）

tkinterで画像を表示するには、まず画像を読み込んでから、ラベルに貼り付ける。

```python
bg_image = tk.PhotoImage(file="omikuji_BG.png")   # 画像を読み込む
bg_label = tk.Label(root, image=bg_image)         # ラベルに画像を乗せる
bg_label.place(x=0, y=0, width=640, height=480)   # ウィンドウ全体に配置
```

#### 🆕 新しい道具その2：`os.path`（画像ファイルの場所を見つける）

実は `file="omikuji_BG.png"` とだけ書くと、**実行する場所によっては画像が見つからずエラー**になることがある。
それを防ぐために、こう書く。

```python
import os

base_dir = os.path.dirname(__file__)                 # このプログラムがあるフォルダ
image_path = os.path.join(base_dir, "omikuji_BG.png")  # そこにある画像のパス
```

> 💡 これは「**画像ファイルの場所を確実に見つけるためのおまじない**」くらいに思っておけばOK。
> 今すぐ完全に理解できなくて大丈夫。気になったら聞いてね。

そして `PhotoImage(file=image_path)` のように、組み立てたパスを渡す。

> ⚠️ 配布フォルダに入っている **`omikuji_BG.png` を、自分のプログラムと同じフォルダに置く**のを忘れずに！

### ③ 部品を「好きな位置」に置く

シンプル版では `pack()`（上から順に自動で並ぶ）を使ったけど、
背景の上に部品を自由に配置したいので、今回は **`place()`** を使う（lesson10でやったね）。

> 📍 **配置の方針：部品（タイトル・説明・ボタン・結果）は、すべて画面の「右半分」に置こう。**
> 背景画像の左側には神様がいるので、その反対側（右半分）の空いているスペースに、縦に並べるイメージ。
> `pack()` ではこの配置はできないので、必ず `place()` で座標を指定すること。
> 細かい位置は自分で調整してOK。実行しながらちょうどいい場所を探してみよう。

#### 🆕 新しい道具その3：`anchor="center"`

`place(x=520, y=60)` だと、部品の**左上**がその座標に来る。
`anchor="center"` をつけると、部品の**中心**がその座標に来るので、位置を合わせやすい。

```python
title_label.place(x=520, y=60, anchor="center")
```

### 配置の目安

下の座標は「右半分に縦並び」にする場合の一例。**この通りでなくてOK**、自分で調整しながらちょうどいい場所を探そう。

| 部品 | x | y |
|------|-----|-----|
| タイトル | 520 | 60 |
| 説明 | 520 | 130 |
| ボタン | 520 | 220 |
| 結果 | 520 | 330 |

> 💡 おみくじを引く処理（パート1で書いた関数）は**そのまま使える**。
> 変えるのは「部品の置き方（`pack` → `place`）」だけ。中身のロジックは完成済み！

### 〔チェックポイント③〕
実行して、背景の上におみくじが表示されたら、今日の目標達成！🎊
ボタンを押して、ちゃんと結果が出るか確認しよう。

# 【パート3】応用編（余力があれば／宿題でもOK）

ここからは自由課題。**授業で時間が余ったらチャレンジ／なければ宿題**でやってみよう。
やり方は説明しません。**どう作るかは自分で考えて、調べて、試してみよう。**
好きなものから、いくつでも挑戦してOK。

## 応用①：結果ごとに色をつける

引いた結果によって、表示される文字の色が変わるようにしよう。

- 「大吉」のとき → 金色
- 「凶」のとき → 青色
- それ以外 → 赤色

## 応用②：確率を操作する

おみくじの「出やすさ」を自分で調整して、**大吉が出やすいおみくじ**を作ってみよう。

逆に「凶だらけの呪いのおみくじ」など、自由に確率をいじって遊んでもOK。

## 応用③：自分だけのおみくじにする

結果の種類を増やしたり、オリジナルの運勢（「超大吉」など）を追加したり、
背景色を結果で変えたり——自由に改造して、自分だけのおみくじアプリにしよう。
