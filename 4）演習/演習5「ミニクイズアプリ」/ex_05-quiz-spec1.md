# 【ステップ１】タプルって何？

## 🆕 新しい道具その1：タプル（tuple）

リストとよく似ているけど、**あとから中身を書き変えられない**「値の組」を作れるのがタプル。

```python
point = ("東京", 35.68, 139.76)   # ( ) で作る（[ ]ではない）

print(point[0])   # "東京"
print(point[1])   # 35.68
```

- 書き方は`[ ]`の代わりに`( )`を使うだけ
- インデックス（`point[0]`など）でのアクセスの仕方はリストと同じ
- 「複数の情報をワンセットで持ち歩きたいけど、書き換える必要はない」ときに向いている

> 💡 なぜ今回タプルを使うか？　クイズの「質問」と「答え」は常にペアで、
> 質問だけ後から書き変える、みたいなことはしないよね。だからタプルがぴったり。

# 【ステップ２】リストの中にタプルを入れる

## 🆕 新しい道具その2：タプルのリスト（複合インデックス）

タプルは、リストの中に複数並べて入れることができる。

```python
questions = [
    ("日本でいちばん高い山は？", "富士山"),
    ("日本の国花は？", "桜"),
    ("日本の首都は？", "東京")
]
```

`questions` は「(質問, 答え)のタプル」が3つ入ったリスト。

```python
print(questions[0])       # ("日本でいちばん高い山は？", "富士山")　← 0番目のタプルまるごと
print(questions[0][0])    # "日本でいちばん高い山は？"           ← 0番目のタプルの0番目（質問）
print(questions[0][1])    # "富士山"                             ← 0番目のタプルの1番目（答え）
```

`[ ]`を2つ重ねて書くのがポイント。**「何番目のタプルか」→「そのタプルの何番目の要素か」**の順番でたどっていく。

### 練習問題


# 【ステップ３】画面の土台と `show_question()`

## 完成イメージ（パート1終了時点）

- ウィンドウを開くと、1問目の質問文が画面に表示される
- **まだ答え合わせボタンは動かない**（次回実装）

## 仕様

### ① ウィンドウ

| 項目 | 値 |
|------|-----|
| タイトル | `ミニクイズアプリ` |
| サイズ | `450x300` |
| サイズ変更 | できないようにする（`root.resizable(False, False)`） |

### ② 画面に置く部品（ウィジェット）

`pack()` を使って、上から順に並べよう。

1. **タイトルのラベル** … 文字「ミニクイズアプリ」／フォントサイズ 18・太字
2. **問題のラベル** … 最初は空文字／フォントサイズ 14／`wraplength=400`、`justify="center"`
3. **入力欄（Entry）** … 幅 25、フォントサイズ 14
4. **ボタン** … 文字「答え合わせ」／フォントサイズ 12（押しても今はまだ何も起きなくてOK）
5. **ボタン** … 文字「次の問題」／フォントサイズ 12（押しても今はまだ何も起きなくてOK）
6. **結果のラベル** … 最初は空文字／フォントサイズ 14

> 💡 `wraplength` は初登場。長い質問文が画面からはみ出さないように、
> 指定した幅で自動的に折り返してくれるオプション。

### ③ クイズのデータと状態を管理する変数

ウィジェットを作る前に、以下の3つを用意しておこう（`root = tk.Tk()` より前）。

```python
questions = [
    ("日本でいちばん高い山は？", "富士山"),
    ("日本の国花は？", "桜"),
    ("日本の首都は？", "東京")
]

current_question = 0   # 今何問目か（0が1問目）
score = 0               # 正解数
```

> 💡 `current_question` と `score` は、関数の外に置く「土台となる変数」。
> なぜ関数の中に書かないのか？　→ 複数の関数（今日の`show_question`、次回の`check_answer`や`next_question`）が
> **同じ値を共有して読み書きする必要がある**から。関数の中に書いてしまうと、その関数の外からは見えなくなってしまう。

### ④ `show_question()` を実装しよう

やること：`question_label` に、今何問目かに応じた質問文を表示する。

- `current_question` が0なら`questions[0][0]`、1なら`questions[1][0]`…という質問文を表示する
- 複合インデックスを使えば、`current_question`の値に応じて自動で正しい質問が取れるはず

最後に、ウィンドウを作ったあと（`root.mainloop()`の前）で `show_question()` を1回呼び出しておこう。
これで起動時に1問目が表示される。

### 〔チェックポイント②〕
実行してみよう。ウィンドウが開いた瞬間に「日本でいちばん高い山は？」が表示されていればOK！🎉
ボタンを押しても何も起きないのは今日のところは正常です。
