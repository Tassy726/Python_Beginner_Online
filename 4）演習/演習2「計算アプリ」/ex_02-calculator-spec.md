#### 🆕 新しい道具その2：`.get()`（入力欄の値を取り出す）

`Entry` に入力された文字は `.get()` で取り出せる。

```python
value = entry1.get()   # entry1 に入力されている文字を取り出す
```

ただし、`.get()` で取り出した値は**文字列（str型）**。
数値として計算したいので、`float()` で変換する必要がある。

```python
num1 = float(entry1.get())   # 入力値をfloat型に変換してから使う
```

> 💡 結果の表示は `result_label.config(text=f"答え：{result}")` のようにf文字列を使おう。

### ④ エラー対策の仕組みを知っておこう（実装は宿題）

チェックポイント②で試したように、数字以外を入力するとプログラムがクラッシュしてしまう。
これを防ぐのが `try/except`。今日は仕組みを頭に入れておこう。

#### 🆕 新しい道具その3（予告）：`try/except`

`try/except` は「エラーが出たときの受け皿」。

```python
def calculate():
    try:
        # ここに「うまくいく場合」の処理を書く
        num1 = float(entry1.get())
        num2 = float(entry2.get())
        result = num1 + num2
        result_label.config(text=f"答え：{result}")
    except ValueError:
        # float()の変換に失敗したとき（文字が入力されたとき）ここが実行される
        result_label.config(text="エラー：数字を入力してください")
```

**`try` ブロック**：まずここを実行しようとする。  
**`except ValueError`**：`float()` で変換できない文字が入っていたとき、ここが実行される。

> 💡 `except` に書く `ValueError` は「値の種類が合わなかった」というエラーの名前。
> `float("abc")` のように、数字に変換できない文字を渡すと、このエラーが発生する。

実際に `calculate()` 関数に組み込む作業は、宿題でやってみよう。
