### ② 画面に置く部品（ウィジェット）

`pack()` を使って、上から順に並べよう。

1. **タイトルのラベル** … 文字「カンタン計算アプリ２」／フォントサイズ 16
2. **ラベル** … 文字「1つ目の数字」／フォントサイズ 12
3. **入力欄（Entry）** … 幅 20、フォントサイズ 12

6. **入力欄（Entry）** … 幅 20、フォントサイズ 12
7. **ボタン** … 文字「計算する」／フォントサイズ 14
8. **結果のラベル** … 文字「答え：」／フォントサイズ 14

#### 🆕 新しい道具その1：`Radiobutton`（ラジオボタン）

複数の選択肢の中から**1つだけ**選べるボタン。
`variable` に「選択状態を記憶する仕組み」を、`value` に「そのボタンを選んだときに入る値」を指定する。

```python
rb_add = tk.Radiobutton(
    root,               # 配置するウィンドウ
    text="＋",          # ボタンに表示する文字
    variable=sel,       # 選択状態を記憶する仕組み（StringVar）
    value="+",          # このボタンを選んだとき sel に入る値
    font=("Arial", 14)  # フォント
)
rb_add.pack()
```

> ⚠️ 4つのラジオボタンに**同じ `variable`** を指定することで、「1つだけ選べる」動作になる。
> `value` がそれぞれ違うことも重要！

ここで疑問が出てくる。**`variable` に何を渡せばいい？**
通常の変数（`a = 1` のような変数）では、ラジオボタンの選択状態をうまく管理できない。
そこで登場するのが `StringVar`。

#### 🆕 新しい道具その2：`StringVar`（選択状態を記憶する仕組み）

tkinterのウィジェットは、Pythonのコードとは**別の世界（C言語で動くtkエンジン）**で動いている。
ラジオボタンを「選んだ」という情報はそのtk側で発生するため、普通のPython変数（`a = 1`）ではその変化を受け取ることができない。

`StringVar` はPython側とtkエンジン側を**つなぐ専用の仕組み**。
ラジオボタンの選択状態が変わると、自動的に中身が書き変わる。

```python
sel = tk.StringVar()   # 選ばれた演算子を記憶する仕組みを作る
sel.set("+")           # 起動時は「＋」を選んだ状態にしておく
```

| メソッド | 意味 |
|--------|------|
| `.set(値)` | `StringVar` に値を書き込む |
| `.get()` | `StringVar` から値を取り出す |

> 💡 `sel.set("+")` と書くことで、アプリ起動時に「＋」が最初から選ばれた状態になる。
> `.get()` は `Entry` でも使ったね。`StringVar` でも同じように「中身を取り出す」命令として使える。
> これを `Radiobutton` の `variable=sel` に渡すことで、選択状態が自動的に同期される。

ラジオボタンが4つ表示されて、1つだけ選べる状態になればOK！

# 【パート2】`Frame` でレイアウトを整える

チェックポイント①で「計算する」ボタンや結果ラベルが消えてしまったのは、ラジオボタンを縦に4つ並べたことで**残りスペースがなくなってしまった**から。
`Frame` を使ってラジオボタンを横並びにすることで解決しよう。

#### 🆕 新しい道具その3：`Frame`（ウィンドウの中のミニウィンドウ）

`Frame` は `root` というウィンドウの中に、さらに**入れ子でミニウィンドウを作る**イメージ。
`Frame` の中に部品を入れることで、その中だけ独立したレイアウトのルールが適用される。

```
┌─── root ────────────────┐
│    カンタン計算アプリ２    │
│      1つ目の数字          │
│      [  入力欄  ]         │
│  ┌─── radio_frame ────┐  │
│  │  [＋][－][×][÷]   │  ← ミニウィンドウ（Frame）
│  └───────────────────┘  │
│      2つ目の数字          │
│      [  入力欄  ]         │
│       [ 計算する ]        │
│          答え：           │
└─────────────────────────┘
```

`root` の中では `pack()` で縦に積み上げ、`radio_frame` の中では `side="left"` で横に並べる。
それぞれのウィンドウの中でレイアウトのルールが**独立している**のがポイント。

```python
radio_frame = tk.Frame(root)    # rootの中にミニウィンドウを作る
radio_frame.pack(pady=8)        # ミニウィンドウをrootの中央に配置する

# ラジオボタンは root ではなく radio_frame の中に作る
rb_add = tk.Radiobutton(radio_frame, text="＋", variable=sel, value="+", font=("Arial", 14))
rb_sub = tk.Radiobutton(radio_frame, text="－", variable=sel, value="-", font=("Arial", 14))
rb_mul = tk.Radiobutton(radio_frame, text="×", variable=sel, value="*", font=("Arial", 14))
rb_div = tk.Radiobutton(radio_frame, text="÷", variable=sel, value="/", font=("Arial", 14))

# radio_frameの中でside="left"で横並びにする
rb_add.pack(side="left", padx=6)
rb_sub.pack(side="left", padx=6)
rb_mul.pack(side="left", padx=6)
rb_div.pack(side="left", padx=6)
```

> 💡 パート1ではラジオボタンを縦に4つ並べたので残りスペースがなくなってしまった。
> `radio_frame` の中で `side="left"` にすることで、**ミニウィンドウの中だけで横並び**になり、
> そのミニウィンドウごと `root` の中央に配置されるのできれいに整う。

### 〔チェックポイント②〕
ラジオボタンが中央にきれいに横並びになればパート2は完成！🎉

# 【宿題】`calculate()` 関数を完成させよう

パート1・2で画面の部品はすべて揃った。
あとは「計算する」ボタンを押したときに**実際に計算する関数**を実装しよう。

## 仕様

- 2つの入力欄から数値を取り出して、選ばれた演算子で計算する
- 結果を結果ラベルに表示する（正常時は `fg="black"`）
- 文字を入力したとき → 結果ラベルに「エラー：数字を入力してください」（`fg="red"`）

## 関数の中でやること

1. `op = sel.get()` で、選ばれた演算子を取得する
   - `"+"` `"-"` `"*"` `"/"` のどれかが返ってくる
2. `if / elif` で `op` の値に応じて計算を分岐させる
   - `op` が `"+"` のとき → 足し算
   - `op` が `"-"` のとき → 引き算
   - … 掛け算・割り算も同様に
3. 結果を結果ラベルに表示する

---

> 📌 **おまけ：`ZeroDivisionError` について**
>
> `÷` を選んで2つ目の数字に `0` を入力すると、プログラムがクラッシュする。
> これは「0で割る」という計算エラーで、`ZeroDivisionError` という名前がついている。
> 余力があれば `except ZeroDivisionError:` を追加して、
> 結果ラベルに「エラー：0で割ることはできません」と表示されるようにしてみよう。
>
> ```python
>     except ValueError:
>         result_label.config(text="エラー：数字を入力してください", fg="red")
>     except ZeroDivisionError:
>         result_label.config(text="エラー：0で割ることはできません", fg="red")
> ```
>
> `except` は何個でも並べて書ける。エラーの種類ごとに対応を変えられるよ。
