### ② 画面に置く部品（ウィジェット）

#### 🆕 新しい道具その1：`Entry` の `show` オプション

`show` オプションを付けると、入力した文字を別の記号に置き換えて表示できる。
パスワード入力欄でおなじみの「●●●●」の仕組み。

```python
entry = tk.Entry(root, width=25, font=("Arial", 14), show="*")
```

> 💡 `show="*"` にすると、何を打っても画面上は `*` で隠れる。
> `show=""`（空文字）にすると、元の文字がそのまま見えるようになる。

### 〔チェックポイント①〕
ここまで書けたら一度実行しよう。
入力欄に文字を打つと `*` で隠れて表示されればOK！
**まだボタンを押しても何も起きない。**

### ③ 「表示」ボタンを押したときの動き

ボタンを押すたびに、パスワードが「見える状態」と「隠れる状態」を交互に切り替える。

やりたいこと：
- 今、表示中か非表示中かを覚えておく
- 押すたびに `entry` の `show` を切り替える（`"*"` ⇔ `""`）
- ボタンの文字も合わせて切り替える（「表示」⇔「非表示」）

#### 🆕 新しい道具その2：`global`（グローバル変数）

**なぜ関数の外に変数を置くのか**

「今、表示中か非表示中か」という情報は、ボタンを押した瞬間だけ分かればいいのではなく、
**次にボタンを押されるまでの間、ずっと覚えておく**必要がある。

もし試しに `is_visible` を関数の中に置くとどうなるか考えてみよう。

```python
def toggle_password():
    is_visible = False   # 関数の中に置いた場合
    ...
```

関数は、呼ばれるたびに**最初から**実行される。つまりボタンを押すたびに
`is_visible = False` からやり直しになり、関数が終わると`is_visible`ごと消えてしまう。
これでは「前回どっちの状態だったか」を覚えておけず、何回押しても同じ動きになってしまう。

状態を次回の呼び出しまで覚えておきたいときは、変数を関数の**外側**に置く必要がある。

```python
is_visible = False   # 関数の外側で用意した変数（プログラムが動いている間、消えずに残り続ける）
```

**ところが、関数の外側の変数は、関数の中から書き換えようとするとエラーになる**

```python
def toggle_password():
    if is_visible:               # ← ここでエラーになる
        ...
        is_visible = False
    else:
        ...
        is_visible = True
```

Pythonは関数を実行する前に、関数の中身全体を確認して
「この関数の中でどこかに代入されている変数は、ローカル変数として扱う」と決めてしまう。
上のコードは `is_visible = False` や `is_visible = True` という代入が中にあるため、
Pythonは `is_visible` を「この関数専用のローカル変数」として扱おうとする。

ところが `if is_visible:` の時点では、そのローカル変数にはまだ何も代入されていない
（代入はif/elseの中でこれから起きる予定）。その結果、

```
UnboundLocalError: cannot access local variable 'is_visible' where it is not associated with a value
```

というエラーが出て、ボタンを押した瞬間にクラッシュしてしまう。

**`global` で解決する**

外側の変数を関数の中から書き換えたいときは、`global` を使って
「この変数は外側にあるあの変数のことだよ、関数専用のローカル変数は作らないよ」と宣言する。

```python
def toggle_password():
    global is_visible   # 外側のis_visibleを操作すると宣言

    if is_visible:
        entry.config(show="*")      # 隠す
        toggle_button.config(text="表示")
        is_visible = False
    else:
        entry.config(show="")       # 見せる
        toggle_button.config(text="非表示")
        is_visible = True
```

> 💡 `global` を書き忘れると「切り替わらない」のではなく「`UnboundLocalError`でクラッシュする」。
> 一度エラーを見ておくと、「読む前に代入がある変数はローカル扱いになる」という
> Pythonのスコープの仕組みが実感として理解しやすくなる。

### 〔チェックポイント②〕

# 【パート2】パスワードの中身を1文字ずつ調べる

ここからは「判定する」ボタンを押したときの処理を作っていく。
まずは以下の2つの部品を追加しよう。

5. **ボタン** … 文字「判定する」／フォントサイズ 14
6. **結果のラベル** … 最初は文字なし／フォントサイズ 14

「判定する」ボタンには関数を1つ作って `command` に結びつけよう。

## やりたいこと

入力されたパスワードの中に、以下の4種類がそれぞれ含まれているかを調べる。

| 種類 | 例 |
|------|-----|
| 小文字 | a, b, c... |
| 大文字 | A, B, C... |
| 数字 | 0, 1, 2... |
| 記号（それ以外） | !, @, # ... |

#### 🆕 新しい道具その3：文字列に対する `for` 文

これまでの `for` は `range()` を使って「決まった回数」くり返していた。
実は `for` は文字列に対しても使えて、その場合は**先頭から1文字ずつ**取り出してくれる。

```python
password = "abc123"

for char in password:   # 文字列を1文字ずつ取り出してcharに代入
    print(char)          # a → b → c → 1 → 2 → 3 の順で表示される
```

#### 🆕 新しい道具その4：文字の判定メソッド

1文字が「小文字か」「大文字か」「数字か」を調べるメソッドがある。

| メソッド | 意味 |
|---------|------|
| `.islower()` | 小文字なら `True` |
| `.isupper()` | 大文字なら `True` |
| `.isdigit()` | 数字なら `True` |

```python
"a".islower()   # True
"A".isupper()   # True
"5".isdigit()   # True
```

> 💡 「小文字でも大文字でも数字でもない」ら、それは記号だと判断できる。
> `if / elif / else` を組み合わせれば、4種類に振り分けられそう。

## 準備するもの：4つの「見つかったよフラグ」

4種類それぞれについて、「1回でも見つかったらTrueにする」ための変数を、
関数の中に4つ用意しておこう（最初は全部 `False`）。

```python
has_lower = False
has_upper = False
has_digit = False
has_symbol = False
```

`for` でパスワードを1文字ずつ調べながら、該当する種類のフラグを `True` にしていけば、
ループが終わったときに「どの種類が含まれていたか」がすべて分かる。

### 〔チェックポイント③〕
`for` と `if/elif/else` と4つのフラグを組み合わせて、
パスワードの中に何が含まれているかをチェックする処理を書いてみよう。
（このチェックポイントでは、`print()` で `has_lower` などの中身を確認してみるとよい）
4種類のフラグが正しく `True` / `False` になっていればパート2は完成！🎉

# 【宿題】パスワードの「強さ」を判定して表示しよう

パート2で、4つのフラグ（`has_lower` / `has_upper` / `has_digit` / `has_symbol`）が
正しく判定できるようになった。あとはその結果をもとに、最終的な強さを判定して画面に表示しよう。

#### 🆕 新しい道具その5：`len()`（文字数を数える）

`len()` は、文字列や リストの「長さ（要素の数）」を調べる関数。

```python
password = "abc123"
len(password)   # 6（文字数が返ってくる）
```

> 💡 リストに対しても使える：`len([1, 2, 3])` は `3`。
> 今回はパスワードの文字数が6文字以上あるかどうかのチェックに使う。

## 仕様

以下の3パターンで判定し、`result_label` に表示する（`fg` で文字色も変える）。

| 条件 | 表示する文字 | 文字色（`fg`） |
|------|-------------|----------------|
| パスワードの文字数が6文字未満 | `NG：6文字以上にしてください` | `red` |
| 4種類（小文字・大文字・数字・記号）が**すべて**含まれている | `強いパスワードです！` | `green` |
| それ以外 | `もう少し強くしましょう` | `orange` |

- 文字数は `len(password)` で調べられる
- 「すべて含まれている」は `if has_lower and has_upper and has_digit and has_symbol:` のように
  `and` でつなげば判定できる
- `if / elif / else` の順番に注意（文字数チェックを一番先にすること）
